    Transformationen

    Dieses Kapitel gibt eine Übersicht über Literalfunktionen, welche Literalwerte ändern oder ersetzen. Wie die im nächsten Kapitel besprochenen Filterfunktionen werden sie innerhalb des <data>-Elementes eingefügt und können, wo sinnvoll, beliebig miteinander kombiniert werden. Dabei wird der Wert von oben nach unten von der einen zur nächsten Funktion durchgereicht. Ein Beispiel:

    <data source="litA" name="gruss">
  <trim/>
  <whitelist>
    <entry name="hallo"/>
    <entry name="grüezi"/>
  </whitelist>
  <case to="upper"/>
</data>

    compose

    compose fügt dem Wert ein Prä- (prefix=) und/oder ein Suffix (postfix=) hinzu

    Beispiel

    Eingabe

    {litA: zeug}

    Morph-Definition

    <data source="litA">
  <compose prefix="be" postfix="ung"/>
</data>

    Ausgabe

    {litA: bezeugung}

    constant

    constant ersetzt Wert durch ein Literal (value=)

    Beispiel

    Eingabe

    {litA: sinnlos}

    Morph-Definition

    <data source="litA">
  <constant value="sinnvoll"/>
</data>

    Ausgabe

    {litA: sinnvoll}

    timestamp

    timestamp ersetzt Wert durch aktuellen Zeitstempel

    • format: Format des Zeitstempels gemäss java.text.SimpleDateFormat
    • timezone: Zeitzone gemäss java.util.TimeZone (Standard: UTC)
    • language: Gebietsschema gemäss java.util.Locale
    Beispiel

    Eingabe

    {}

    Morph-Definition

    <data source="_id" name="zeitstempel">
  <timestamp format="yyyy-MM-dd HH:mm:ss" timezone="GMT+02:00" language="de"/>
</data>

    Ausgabe

    {zeitstempel: 2018-08-24 17\:59\:18}

    substring

    substring extrahiert Substring basierend auf Indizes (start=, end=)

    Beispiel

    Eingabe

    {litA: bezeugung}

    Morph-Definition

    <data source="litA">
  <substring start="2" end="6"/>
</data>

    Ausgabe

    {litA: zeug}

    regexp

    regexp extrahiert einen Substring basierend auf einem regulären Ausdruck (match=). Rückwärtsreferenzen (capture groups) werden unterstützt, welche durch format="${1..n}" wiederverwendet werden können. Syntax entspricht derjenigen von regulären Ausdrücken in Java.

    Beispiel

    Eingabe

    {litA: eins zwei drei}

    Morph-Definition

    <data source="litA">
  <regexp match="\S+ (\S+) \S+" format="${1} bücher"/>
</data>

    Ausgabe

    {litA: zwei bücher}

    replace / setreplace

    replace / setreplace ersetzt einen passenden Substring (pattern=) durch einen anderen Wert (with=). Java-konforme reguläre Ausdrücke werden unterstützt. Durch setreplace lässt sich eine ganze lokale Ersetzungstabelle definieren (<entry name="ausdruck" value="ersetzung"/>) oder mithilfe von map= eine solche referenzieren.

    Für weitere Informationen siehe Abschnitt “Nachschlagetabellen”

    Beispiel

    Eingabe

    {litA: eins, litA: zwei, litA: drei, litA: vier}

    Morph-Definition

    <data source="litA" name="zahl">
  <setreplace>
    <entry name="eins" value="1"/>
    <entry name="zwei" value="2"/>
    <entry name="drei" value="3"/>
    <entry name="vier" value="4"/>
  </setreplace>
</data>

    Ausgabe

    {litA: 1, litA: 2, litA: 3, litA: 4}

    lookup

    lookup ist analog zu setreplace, unterstützt aber nur statische Strings als Suchmuster. Mithilfe von in= kann alternativ auch eine Konkordanztabelle referenziert werden. default= definiert einen Standardwert, wenn Schlüssel nicht in Tabelle definiert ist.

    Für weitere Informationen siehe Abschnitt “Nachschlagetabellen”

    Beispiel

    Eingabe

    {kanton: Waadt, kanton: Wallis, kanton: Jura, kanton: Genf}

    Morph-Definition

    <data source="kanton" name="hauptort">
  <lookup default="nicht bekannt">
    <entry name="Wallis" value="Sion"/>
    <entry name="Jura" value="Delémont"/>
    <entry name="Waadt" value="Lausanne"/>
  </lookup>
</data>

    Ausgabe

    {hauptort: Lausanne, hauptort: Sion, hauptort: Delémont, hauptort: nicht bekannt}

    case

    case wandelt Wert in Gross- (upper) bzw. Kleinbuchstaben (lower) (to=) um. Nimmt optional ein Sprach-Tag entgegen (language=)

    Beispiel

    Eingabe

    {buch: kitabı}

    Morph-Definition

    <data source="buch">
  <case to="upper" language="tr"/>
</data>

    Ausgabe

    {buch: KİTABI}

    trim

    trim schneidet Leerzeichen am Ende des Feldwertes ab

    Beispiel

    Eingabe

    {litA: 'luftig   '}

    Morph-Definition

    <data source="litA">
  <trim />
</data>

    Ausgabe

    {litA: luftig}

    normalize-utf8

    normalize-utf8 normalisiert Zeichen in kanonische Form, bspw. ein ä, das aus dem Diakrit ¨ + a gebildet wurde, zu ä. Für Details siehe hier.

    Beispiel

    Eingabe

    {litA: ä }

    Morph-Definition

    <data source="litA">
  <normalize-utf8 />
</data>

    Ausgabe

    {litA: ä}

    dateformat

    dateformat formatiert Datum

    • inputformat: Format des Inputdatums gemäss java.text.SimpleDateFormat
    • outputformat: Format des Ausgabedatums: FULL, LONG (Standard), MEDIUM oder SHORT (s. Beschreibung in java.text.DateFormat)
    • era: Epoche: AUTO (Standard), AD oder BC
    • removeLeadingZeros: Entferne führende Nulle (Standard: false)
    • language: Gebietsschema gemäss java.util.Locale

    removeLeadingZeros="true" entfernte in unseren Tests fälschlicherweise auch Nulle in vierstelligen Jahreszahlen, das Resultat war also bspw. 218 anstelle von 2018!

    Beispiel

    Eingabe

    {heute: 2018-08-26 }

    Morph-Definition

    <data source="heute">
  <dateformat inputformat="yyyy-MM-dd" outputformat="FULL" era="AD" removeLeadingZeros="false" language="de"/>
</data>

    Ausgabe

    {heute:Sonntag\, 26. August 2018}

    split

    split teilt Feldwert basierend auf regulärem Ausdruck (delimiter=) in Substrings

    Beispiel

    Eingabe

    {heute: 2018-08-26 }

    Morph-Definition

    <data source="heute" name="monat">
  <split delimiter="-"/>
  <occurrence only="moreThan 1"/>
  <occurrence only="lessThan 2"/>
</data>

    occurrence ist ein Filter, welche nur Werte weiterleitet, die bis zu oder ab einem bestimmten Index in einer Liste von Werten vorkommen.

    Ausgabe

    {monat: 10}

    isbn

    isbn transformiert ISBN-Nummern. Das Attribut to akzeptiert drei Werte: isbn10 für eine Umwandlung von ISBN13 zu ISBN10, isbn13 für das Umgekehrte und clean für eine Bereinigung der Nummer. Optional können zudem die Attribute verifyCheckDigit, standardmässig false, und errorString gesetzt werden.

    Beispiel

    Eingabe

    {isbn: 978-3-89401-810}

    Morph-Definition

    <data source="isbn">
  <isbn to="isbn10" verifyCheckDigit="true" errorString="ungültig"/>
</data>

    Ausgabe

    {isbn:ungültig}

    urlencode

    urlencode enkodiert Wert als URL

    Beispiel

    Eingabe

    {suchwert: gelebtes leben}

    Morph-Definition

    <data source="suchwert">
  <urlencode />
  <compose prefix="https://swissbib.ch/Search/Results?lookfor="/>
</data>

    Ausgabe

    {suchwert:https\://swissbib.ch/Search/Results?lookfor=gelebtes+leben}

    htmlanchor

    htmlanchor baut aus Wert einen HTML-Link. Umfasst drei Attribute: prefix (zwingend), postfix und title (optional)

    Beispiel

    Eingabe

    {artikel: mein interessanter beitrag}

    Morph-Definition

    <data source="anker">
  <urlencode />
  <htmlanchor prefix="https://example.com/" title="Mein interessanter Beitrag"/>
</data>

    Ausgabe

    {anker:<a href="https\://example.com/mein+interessanter+beitrag">Mein interessanter Beitrag</a>}

    switch-name-value

    switch-name-value vertauscht Feldname und -wert

    Beispiel

    Eingabe

    {litA: wertA}

    Morph-Definition

    <data source="litA">
  <switch-name-value/>
</data>

    Ausgabe

    {wertA: litA}

    script

    script übergibt Wert zur Transformation an eine Javascript-Funktion (invoke=) in einer externen Datei (file=)

    Hinweise zur Implementierung einer solchen Funktion finden sich im Anhang

    Beispiel

    Eingabe

    {palindrom: Regallager}

    Morph-Definition

    <data source="litA">
  <script file="stringutils.js" invoke="reverse"/>
</data>

    stringutils.js, das eine Funktion reverse enthält, muss sich in diesem Fall im gleichen Verzeichnis befinden.

    Ausgabe

    {palindrom: regallageR}

    java

    java übergibt Wert zur Tranformation einer Java-Klasse (class=, voller Name). Werte aller weiteren Attribute werden einer gleichnamigen setter-Methode der Klasse übergeben.

    Hinweise zur Implementierung einer solchen Klasse finden sich im Anhang

    Beispiel

    Eingabe

    {dekodiert: Eine strenggeheime Botschaft!}

    Morph-Definition

    <data source="dekodiert" name="enkodiert">
  <java class="com.example.secret.Encoder" algorithm="ROT13" />
</data>

    Damit das Beispiel funktioniert, müsste ein Metafacture-Plugin die Klasse Encoder im angegebenen Klassenpfad zur Verfügung stellen. Zudem muss die Klasse eine setter-Methode setAlgorithm haben.

    Ausgabe

    {enkodiert: Rvar fgerattrurvzr Obgfpunsg!}

    count

    count gibt Anzahl der aktuell gezählten Werte aus.

    Um nur den letzten Wert zu berücksichtigen, siehe das Rezept “Nur letzten Wert eines Literals berücksichtigen”.

    Beispiel

    Eingabe

    {litA: wert1, litA: wert2}

    Morph-Definition

    <data source="litA">
  <count/>
</data>

    Ausgabe

    {litA: 1, litA: 2}
